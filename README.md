# Интеллектуальная система рекомендаций QoP для VoIP

Система адаптивных рекомендаций по уровню защиты (Quality of Protection) для телекоммуникационных сетей на базе Asterisk PBX. Использует гибридную ML-модель (XGBoost + PyTorch MLP) для анализа телеметрии в реальном времени и формирования рекомендаций по балансу между безопасностью и качеством связи.

## Уровни рекомендаций QoP

| Уровень | Профиль | Условия | Описание |
|---------|---------|---------|----------|
| **Low** | SIP + RTP | LAN, идеальный канал | Без шифрования — минимальная нагрузка |
| **Medium** | SIP TLS | WAN, плохой канал | Шифрование сигнализации, открытый голос |
| **High** | SIP TLS + SRTP | WAN, хороший канал | Полное сквозное шифрование |

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Обучение модели

```bash
python train_model.py
```

Автоматически генерирует датасет (9900+ сэмплов) и обучает гибридную модель.

### 3. Запуск демонстрации (без Asterisk)

```bash
python main.py demo
```

Симулирует 3 вызова (LAN, Wi-Fi, 4G), через 15 секунд вносит аномалию, демонстрирует переключение рекомендаций.

### 4. Интерактивный режим

```bash
python main.py scenario
```

Позволяет вручную вносить аномалии и наблюдать реакцию ML-агента.

### 5. Подключение к реальному Asterisk

```bash
# Настроить config.yaml (host, порт, логин AMI)
python main.py live
```

## Развертывание Asterisk

### Сервер (Ubuntu)

```bash
sudo apt install asterisk
# Скопировать конфиги
sudo cp asterisk_config/*.conf /etc/asterisk/
sudo systemctl restart asterisk
```

### Клиенты

- **LAN**: Zoiper/MicroSIP, сервер = внутренний IP
- **WAN**: Zoiper (мобильный), сервер = публичный IP, порт 5060

### Порты для проброса

- **5060/UDP** — SIP-сигнализация
- **5061/TCP** — SIP TLS
- **10000-20000/UDP** — RTP (голос)

## Стресс-тестирование (Этап 5)

На сервере Asterisk:

```bash
# Имитация деградации 4G
sudo tc qdisc add dev eth0 root netem delay 150ms 30ms loss 5%

# Снятие аномалии
sudo tc qdisc del dev eth0 root
```

В демо-режиме аномалии вносятся программно через симулятор.

## ML-модель

### Архитектура

**Stage 1 — XGBoost (Gradient Boosting):**
- Входные признаки: latency, jitter, packet_loss, is_external, samples_count
- Выход: вероятности 3 классов QoP

**Stage 2 — MLP (PyTorch Neural Network):**
- Вход: вероятности XGBoost (3) + исходные признаки (5) = 8 признаков
- Архитектура: 64 → 32 → 16 → 3 нейрона
- BatchNorm + Dropout для регуляризации
- Выход: финальная классификация QoP

### Датасет

- 9000 основных сэмплов (по 3000 на класс)
- 900 граничных кейсов для робастности
- Разделение: 80% train / 20% test
